# KoinoriBot NoneBot2 项目说明文档

本文档详细说明 KoinoriBot NB2 的架构设计及各模块的迁移状态。

## 1. 核心架构说明

本机器人采用 [NoneBot2](https://nonebot.dev/) 框架重构，旨在实现对 OneBot V11 和官方 QQBot 的双协议支持。

### UID 兼容层 (`uid_manager.py`)
为了解决不同平台用户 ID 格式不一致的问题（OneBot 使用 QQ 号整数，QQBot 使用 OpenID 字符串），项目实现了统一的 UID 系统：
- **`uid_manager`**：负责维护 `platform + external_id` 到内部 `uid` (int) 的映射。
- **自动映射**：所有命令通过 `event_adapter` 调用时，会自动查找或创建对应的 `uid`。
- **优势**：业务逻辑（如金币、背包）只需记录整数 `uid`，无需关心底层协议。

### 事件适配器 (`event_adapter.py`)
为了让一套业务代码同时运行在不同协议上，项目封装了统一事件接口：
- **`UnifiedEvent`**：封装了 `uid`, `message_text`, `nickname` 等通用属性。
- **`adapt_event(event, bot)`**：将 `OneBotV11Event` 或 `QQBotEvent` 转换为 `UnifiedEvent`。
- **`send_message`**：统一的发信接口，根据事件来源自动选择正确的发送方式（如 OneBot 的 `send_msg` 或 QQBot 的 API）。
